# 📘 MO端模块（成员 D）README

## 📌 项目简介

本模块属于项目中的 **MO端（岗位 + 模块 + 工时）管理系统**，主要面向管理员（Admin）与学生用户，用于完成岗位发布、审批流程、工时管理及绩效评价等核心业务。

本 README 主要说明本人（成员 D）在项目中负责的功能模块、设计思路及实现要点。

---

## 🧩 功能模块总览

本人负责以下 9 个核心模块：

1. 发布岗位
2. 岗位状态管理
3. 审批流程展示
4. 催促功能
5. 我的模块
6. 模块变更申请
7. 工时审批
8. 绩效评分
9. 工时统计

---

## 🚀 详细功能说明

### 1️⃣ 发布岗位

**功能描述：**
用于管理员创建新的岗位信息。

**核心功能：**

* 新建岗位表单

  * 模块
  * 角色
  * 名额
  * 工时
  * 截止日期
  * 描述
  * 要求
* 支持保存草稿
* 提交审核
* 编辑草稿

**设计要点：**

* 表单校验（必填项、格式校验）
* 草稿与正式提交状态区分
* 提交后进入审批流程

---

### 2️⃣ 岗位状态管理

**功能描述：**
用于查看岗位在不同阶段的状态。

**核心功能：**

* 四状态 Tab：

  * 草稿
  * 待审
  * 已发布
  * 已完成
* 状态徽章展示
* 申请人数统计
* 截止日期倒计时

**设计要点：**

* 状态驱动 UI 渲染
* 倒计时实时更新

---

### 3️⃣ 审批流程展示

**功能描述：**
展示岗位从提交到审批完成的完整流程。

**核心功能：**

* 工作流横幅（5步骤可视化）
* 当前状态高亮
* 待审批提示

**设计要点：**

* 流程组件复用
* 状态与流程节点绑定

---

### 4️⃣ 催促功能

**功能描述：**
用于提醒管理员或审批人尽快处理任务。

**核心功能：**

* 催促按钮（Admin）
* 30秒冷却倒计时
* 催促次数记录
* Toast 通知提示

**设计要点：**

* 防止频繁触发（节流/防抖）
* 前后端时间同步

---

### 5️⃣ 我的模块

**功能描述：**
展示当前用户参与的模块信息。

**核心功能：**

* 模块卡片展示

  * 代码
  * 标题
  * 状态
  * 学期
* TA名额进度条
* 预算工时进度条
* 预算预警（≥80%）
* 已填/剩余统计

**设计要点：**

* 卡片组件化
* 进度条动态计算

---

### 6️⃣ 模块变更申请

**功能描述：**
用于对已有模块进行修改申请。

**核心功能：**

* 新增/编辑申请表单
* 保存草稿
* 提交审核
* 状态追踪：

  * 草稿
  * 待审
  * 通过
  * 拒绝
* Admin备注展示

**设计要点：**

* 与审批流程复用
* 状态流转清晰

---

### 7️⃣ 工时审批

**功能描述：**
管理员对学生提交的工时进行审批。

**核心功能：**

* 工时卡片展示

  * 学生
  * 模块
  * 周次
  * 工时
  * 任务清单
* 展开/折叠详情
* 批准 / 拒绝操作

**设计要点：**

* 列表性能优化（分页/虚拟列表）
* 审批操作即时反馈

---

### 8️⃣ 绩效评分

**功能描述：**
对学生进行绩效评价。

**核心功能：**

* 1-5 星评分
* 快速评语选择
* 自定义评语输入

**设计要点：**

* 评分组件复用
* 表单与评价数据绑定

---

### 9️⃣ 工时统计

**功能描述：**
对整体工时数据进行汇总分析。

**核心功能：**

* 待审数量
* 待审工时
* 本周已批准工时
* 活跃 TA 数量

**设计要点：**

* 数据聚合计算
* 图表/指标展示（可扩展）

---

## 🏗️ 技术实现要点

* 前端框架：Vue / React（根据项目实际）
* 状态管理：Vuex / Redux
* UI组件：Element UI / Ant Design
* 接口通信：Axios

**通用设计原则：**

* 组件化开发
* 状态驱动 UI
* 可复用性优先
* 清晰的数据流

---

## 🔄 数据流与状态设计

* 岗位生命周期：

  ```
  草稿 → 待审 → 已发布 → 已完成
  ```

* 模块申请状态：

  ```
  草稿 → 待审 → 通过 / 拒绝
  ```
