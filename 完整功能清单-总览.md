# BUPT TA 招聘系统 - 完整功能清单总览

## 项目概述

本项目是一个基于 Java Servlet 的 TA（Teaching Assistant）招聘管理系统，采用 DDD（领域驱动设计）架构，使用嵌入式 HTTP 服务器部署，JSON 文件存储数据。

### 技术栈
- **后端**：Java 17, Jakarta Servlet API, Gson
- **前端**：HTML5, CSS3, JavaScript (原生)
- **服务器**：JDK 内置 HttpServer（嵌入式）
- **数据存储**：JSON 文件
- **架构模式**：DDD（领域驱动设计）、仓储模式、适配器模式

### 系统角色
1. **Student（学生）**：浏览职位、申请职位、提交工时
2. **MO（Module Organizer）**：创建职位、审核申请、审核工时
3. **Admin（管理员）**：系统管理（未实现）

## 功能清单目录

### 第1部分：基础架构
详见：`完整功能清单-1-基础架构.md`

1. **嵌入式服务器**
   - 使用 JDK HttpServer 提供 HTTP 服务
   - 适配器模式桥接 Servlet API
   - 端口：9191

2. **DDD 架构**
   - Interfaces 层（API 接口）
   - Application 层（业务逻辑）
   - Domain 层（领域模型）
   - Infrastructure 层（数据访问）

3. **JSON 文件存储**
   - 使用 Gson 序列化/反序列化
   - 读写锁保证线程安全
   - 6 个数据文件（students, users, jobs, applications, timesheets, modules）

4. **统一响应格式**
   - 标准 JSON 响应：`{code, message, data}`
   - 统一错误处理

5. **Session 管理**
   - 基于 HTTP Session 的用户认证
   - 支持学生和 MO/Admin 两种用户类型

## API 端点汇总

### 认证模块
```
POST   /api/auth/login       - 用户登录
POST   /api/auth/register    - 用户注册
POST   /api/auth/logout      - 用户登出
GET    /api/auth/me          - 获取当前用户信息
```

### 职位模块（公开）
```
GET    /api/jobs             - 获取职位列表
GET    /api/jobs/{id}        - 获取职位详情
```

### 学生端
```
GET    /api/student/profile                    - 获取个人资料
PUT    /api/student/profile                    - 更新个人资料
GET    /api/student/applications               - 获取申请列表
POST   /api/student/applications               - 申请职位
DELETE /api/student/applications/{id}          - 撤回申请
GET    /api/student/timesheets                 - 获取工时列表
POST   /api/student/timesheets                 - 提交工时表
```

### MO端
```
# 职位管理
GET    /api/mo/jobs                  - 获取职位列表
GET    /api/mo/jobs/{id}             - 获取职位详情
POST   /api/mo/jobs                  - 创建职位
PUT    /api/mo/jobs/{id}             - 更新职位
DELETE /api/mo/jobs/{id}             - 删除职位
POST   /api/mo/jobs/{id}/submit      - 发布职位

# 申请人管理
GET    /api/mo/applicants            - 获取申请人列表
GET    /api/mo/applicants/{id}       - 获取申请详情
PUT    /api/mo/applicants/{id}/status - 审核申请

# 工时审核
GET    /api/mo/timesheets            - 获取工时表列表
GET    /api/mo/timesheets/{id}       - 获取工时表详情
PUT    /api/mo/timesheets/{id}/review - 审核工时表

# 课程模块
GET    /api/mo/modules               - 获取课程列表
GET    /api/mo/modules/{id}          - 获取课程详情
```

## 数据模型

### 核心实体

#### Student（学生）
```
- id: 学生ID (S001, S002...)
- name: 姓名
- email: 邮箱
- password: 密码
- studentId: 学号
- phone: 电话
- major: 专业

- status: 状态 (active/inactive)
```

#### User（MO/Admin）
```
- id: 用户ID (MO001, ADMIN001...)
- name: 姓名
- email: 邮箱
- password: 密码
- role: 角色 (mo/admin)
- status: 状态 (active/inactive)
```

#### Job（职位）
```
- id: 职位ID (JOB001, JOB002...)
- moduleId: 课程模块ID
- moId: MO ID
- title: 职位标题
- description: 职位描述
- responsibilities: 职责列表
- requirements: 要求列表
- requiredSkills: 技能要求
- hoursPerWeek: 每周工时
- hourlyRate: 时薪
- schedule: 工作时间
- startDate: 开始日期
- endDate: 结束日期
- slots: 招聘人数
- status: 状态 (draft/published)

```

#### Application（申请）
```
- id: 申请ID (APP001, APP002...)
- studentId: 学生ID
- jobId: 职位ID
- status: 状态 (pending/accepted/rejected)
- coverLetter: 个人陈述
- appliedAt: 申请时间
- reviewedBy: 审核人ID
- reviewedAt: 审核时间
- reviewComment: 审核意见
```

#### Timesheet（工时表）
```
- id: 工时表ID (TS001, TS002...)
- studentId: 学生ID
- jobId: 职位ID
- date: 工作日期
- hours: 工作时长
- description: 工作描述
- status: 状态 (pending/approved/rejected)
- submittedAt: 提交时间
- reviewedBy: 审核人ID
- reviewedAt: 审核时间
- reviewComment: 审核意见
- approvedHours: 批准工时
```

#### CourseModule（课程模块）
```
- id: 模块ID (MOD001, MOD002...)
- name: 课程名称
- code: 课程代码
- department: 院系
- semester: 学期
- moId: 负责人ID
- description: 课程描述
```

## 业务流程

### 学生申请流程
```
1. 学生浏览职位列表
2. 查看职位详情
3. 提交申请（含个人陈述）
4. 等待 MO 审核
5. 查看审核结果
   - 通过：可以开始工作
   - 拒绝：可以申请其他职位
```

### MO 招聘流程
```
1. MO 创建职位（draft）
2. 编辑职位信息
3. 发布职位（published）
4. 查看申请人列表
5. 审核申请（accept/reject）
6. 查看学生提交的工时
7. 审核工时（approve/reject）
```

### 工时管理流程
```
1. 学生提交工时表（pending）
2. MO 查看工时详情
3. MO 审核工时
   - 批准：可调整批准工时数
   - 拒绝：批准工时为 0
4. 学生查看审核结果
```

## 设计模式

### 1. 领域驱动设计（DDD）
- 分层架构：Interfaces, Application, Domain, Infrastructure
- 领域实体：Student, Job, Application, Timesheet
- 仓储模式：Repository 封装数据访问

### 2. 适配器模式
- HttpServletRequestAdapter：适配 HttpExchange 到 HttpServletRequest
- HttpServletResponseAdapter：适配 HttpExchange 到 HttpServletResponse
- HttpSessionAdapter：实现 Session 管理

### 3. 仓储模式
- StudentRepository, JobRepository, ApplicationRepository 等
- 统一的 CRUD 接口
- 隔离业务逻辑和数据存储

### 4. 单例模式
- DataStore：统一的数据存储管理器
- 使用静态方法提供全局访问点

1. **无外部依赖部署**

   - 一键启动脚本

2. **线程安全的数据访问**
   - 读写锁机制
   - 支持并发读取

3. **清晰的架构设计**

4. **完整的测试数据**

   - 覆盖所有业务场景

   - 统一响应格式
   - 清晰的错误处理
