# 数据存储说明

## 存储方式

所有数据以 **JSON 文件** 形式存储在：
```
resources/data/
```

## 数据文件列表

| 文件名 | 内容 | 说明 |
|--------|------|------|
| `students.json` | 学生信息 | 注册的学生账号和个人信息 |
| `users.json` | MO/Admin 账号 | 教职工和管理员账号 |
| `jobs.json` | 职位信息 | 发布的 TA 职位 |
| `applications.json` | 申请记录 | 学生的职位申请 |
| `timesheets.json` | 工时记录 | 学生提交的工时表 |
| `modules.json` | 课程模块 | 课程信息 |

## 注册流程

### 学生注册

1. **前端提交**：学生在 `/login.html` 填写注册表单
2. **后端处理**：`AuthServlet.handleRegister()` 处理请求
3. **数据验证**：
   - 检查邮箱是否已存在
   - 检查学号是否已存在
   - 验证必填字段
4. **生成 ID**：`StudentRepository.generateId()` 生成唯一 ID（如 S001, S002...）
5. **保存数据**：
   - 创建 `Student` 对象
   - 调用 `StudentRepository.save(student)`
   - 写入 `resources/data/students.json`
6. **返回结果**：返回用户信息（不含密码）

### 数据持久化

```java
// 保存流程
StudentRepository.save(student)
  ↓
DataStore.saveStudents(students)
  ↓
JsonFileUtil.writeJson("resources/data/students.json", students)
  ↓
文件写入磁盘
```

## 线程安全

使用 `ReadWriteLock` 保证并发访问安全：
- **读操作**：多个线程可同时读取
- **写操作**：独占锁，确保数据一致性

## 查看数据

```

```

### 方法 2：使用在线 JSON 查看器
1. 复制 JSON 文件内容
2. 访问 https://jsonformatter.org/
3. 粘贴并格式化查看

### 方法 3：通过 API 查询
```javascript
// 浏览器控制台
fetch('http://localhost:9191/api/student/profile')
  .then(r => r.json())
  .then(console.log)
```

## 注意事项

1. **密码明文存储**：当前密码是明文，生产环境需要加密
2. **文件锁定**：服务器运行时不要手动编辑 JSON 文件
3. **数据格式**：手动编辑时确保 JSON 格式正确
4. **ID 唯一性**：系统自动生成 ID，不要手动修改

## 测试注册

1. 访问 http://localhost:9191/login.html
2. 点击 "Create Account"
3. 填写信息并提交
4. 注册成功后，查看 `resources/data/students.json`
5. 新用户数据已添加到文件末尾

## 数据初始化

系统启动时会加载初始测试数据：
- 3 个学生账号
- 2 个 MO 账号
- 1 个管理员账号
- 若干测试职位

初始数据在 `DataInitializer.java` 中定义。
