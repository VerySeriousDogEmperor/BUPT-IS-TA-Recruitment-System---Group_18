# 修复：个人空间数据显示问题

## 问题描述
用户反馈：在个人空间的 Basic Info 页面，注册时填写的信息（如 phone, studentId, major, gpa 等）无法显示，需要重新填写。

## 问题分析

### 根本原因
在 `AuthServlet.java` 的登录和注册接口中，返回给前端的用户数据不完整。

#### 修复前的代码
```java
// 登录时只返回 4 个字段
Map<String, Object> userData = new HashMap<>();
userData.put("id", student.getId());
userData.put("name", student.getName());
userData.put("email", student.getEmail());
userData.put("role", student.getRole());

ResponseUtil.sendSuccess(response, "登录成功", userData);
```

这导致前端 `localStorage` 中只保存了这 4 个字段，而缺少：
- `phone` - 电话号码
- `studentId` - 学号
- `major` - 专业
- `grade` - 年级
- `gpa` - GPA
- `avatar` - 头像
- `skills` - 技能列表
- `resume` - 简历信息
- `schedule` - 课程表

### 数据流程
1. 用户注册时填写完整信息 → 保存到 `students.json`
2. 登录时后端返回不完整数据 → 保存到 `localStorage`
3. 个人空间页面从 `localStorage` 读取 → 显示不完整

## 解决方案

### 修改文件
`src/com/bupt/ta/shared/interfaces/AuthServlet.java`

### 修改内容

#### 1. 修复学生登录接口
```java
// 修改前
Map<String, Object> userData = new HashMap<>();
userData.put("id", student.getId());
userData.put("name", student.getName());
userData.put("email", student.getEmail());
userData.put("role", student.getRole());
ResponseUtil.sendSuccess(response, "登录成功", userData);

// 修改后
student.setPassword(null);  // 不返回密码
ResponseUtil.sendSuccess(response, "登录成功", student);
```

#### 2. 修复学生注册接口
```java
// 修改前
Map<String, Object> userData = new HashMap<>();
userData.put("id", student.getId());
userData.put("name", student.getName());
userData.put("email", student.getEmail());
userData.put("role", student.getRole());
ResponseUtil.sendSuccess(response, "注册成功", userData);

// 修改后
student.setPassword(null);  // 不返回密码
ResponseUtil.sendSuccess(response, "注册成功", student);
```

#### 3. 修复获取当前用户接口 (/api/auth/me)
```java
// 修改前
if ("student".equals(role)) {
    Optional<Student> studentOpt = studentRepo.findById(userId);
    if (studentOpt.isPresent()) {
        Student student = studentOpt.get();
        userData.put("id", student.getId());
        userData.put("name", student.getName());
        userData.put("email", student.getEmail());
        userData.put("role", student.getRole());
    }
}

// 修改后
if ("student".equals(role)) {
    Optional<Student> studentOpt = studentRepo.findById(userId);
    if (studentOpt.isPresent()) {
        Student student = studentOpt.get();
        student.setPassword(null);
        ResponseUtil.sendSuccess(response, student);
        return;
    }
}
```

## 安全性说明

### 密码处理
修改后的代码在返回用户数据前，会将密码字段设置为 `null`：
```java
student.setPassword(null);
```

这确保了密码不会被发送到前端，提高了安全性。

### 注意事项
1. 密码以明文存储在 `students.json` 中（仅用于演示）
2. 生产环境应使用加密存储（如 BCrypt）
3. 建议使用 HTTPS 传输敏感数据

## 后续优化建议

### 1. 统一返回格式
建议为 MO 和 Admin 也返回完整的用户对象，而不是手动构建 Map。

### 2. 添加字段过滤
可以添加一个工具方法来过滤敏感字段：
```java
public static User sanitizeUser(User user) {
    user.setPassword(null);
    // 可以添加更多字段过滤逻辑
    return user;
}
```

### 3. 使用 DTO
创建专门的数据传输对象（DTO）来控制返回的字段：
```java
public class UserDTO {
    private String id;
    private String name;
    private String email;
    // ... 其他需要返回的字段
}
```
