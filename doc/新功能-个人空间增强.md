# 个人空间增强功能

## 更新时间
2026-04-08

## 功能概述
为学生个人空间页面添加了三个重要功能：
1. 头像上传功能
2. GPA 填写提示弹窗
3. 显示后台生成的学生 ID

---

## 1. 头像上传功能

### 功能描述
学生可以上传自己的头像照片，头像会显示在个人空间页面的顶部。

### 使用方法
1. 进入个人空间页面（`/student/dashboard.html`）
2. 将鼠标悬停在头像区域
3. 会出现"Upload"提示
4. 点击头像区域
5. 选择图片文件（支持 jpg, png, gif 等格式）
6. 图片会自动上传并显示

### 技术实现

#### 后端修改
**文件**: `src/com/bupt/ta/student/domain/Student.java`
- 添加了 `avatar` 字段（String 类型）
- 添加了 `getAvatar()` 和 `setAvatar()` 方法

**文件**: `src/com/bupt/ta/student/interfaces/StudentProfileServlet.java`
- 在 `doPut()` 方法中添加了头像更新逻辑
- 支持通过 API 更新头像

#### 前端修改
**文件**: `web/student/dashboard.html`
- 修改了头像容器结构
- 添加了头像图片元素 `<img id="profileAvatarImg">`
- 添加了上传覆盖层 `<div class="avatar-upload-overlay">`
- 添加了文件输入 `<input type="file" id="avatarUpload">`

**文件**: `web/static/js/pages/student/dashboard.js`
- 添加了 `initAvatarUpload()` 函数
- 实现了图片文件读取和 Base64 转换
- 添加了文件类型和大小验证（最大 2MB）
- 更新了 `loadUserData()` 函数以显示头像

**文件**: `web/static/css/dashboard.css`
- 添加了头像上传相关样式
- 实现了悬停效果

### 数据存储
- 头像以 Base64 格式存储在 `localStorage` 中
- 同时会保存到后端的 `students.json` 文件

### 限制
- 图片大小限制：2MB
- 支持的格式：所有浏览器支持的图片格式（jpg, png, gif, webp 等）

## 2. GPA 填写提示弹窗

### 功能描述
当学生首次进入个人空间或 GPA 为空时，会自动弹出提示窗口，要求填写 GPA。

### 触发条件
- 用户的 GPA 为 `null`、`undefined` 或 `0`
- 页面加载后 500ms 自动弹出

### 使用方法
1. 进入个人空间页面
2. 如果 GPA 未填写，会自动弹出弹窗
3. 在输入框中输入 GPA（0.0 - 4.0）
4. 点击"Save GPA"按钮
5. GPA 会保存并显示在页面上

### 技术实现

#### 前端修改
**文件**: `web/student/dashboard.html`
- 添加了 GPA 弹窗 HTML 结构
```html
<div id="gpaModal" class="modal">
    <div class="modal-content">
        <form id="gpaForm">
            <input type="number" id="gpaInput" min="0" max="4" step="0.01">
        </form>
    </div>
</div>
```

**文件**: `web/static/js/pages/student/dashboard.js`
- 添加了 `showGPAModal()` 函数
- 添加了 `hideGPAModal()` 函数
- 添加了 `initGPAModal()` 函数
- 在 `loadUserData()` 中添加了 GPA 检查逻辑

**文件**: `web/static/css/dashboard.css`
- 添加了模态框样式
- 添加了淡入淡出动画

### 验证规则
- GPA 必须是数字
- 范围：0.0 - 4.0
- 支持两位小数

### 数据存储
- GPA 保存到 `localStorage`
- 同时会保存到后端的 `students.json` 文件

## 3. 显示学生 ID

### 功能描述
在个人空间页面顶部显示后台自动生成的学生 ID（如 S001, S002 等），而不是显示 "N/A"。

### 显示位置
- 页面顶部的个人信息区域
- "GPA" 标签旁边
- 格式：`ID: S001`

### 技术实现

#### 前端修改
**文件**: `web/static/js/pages/student/dashboard.js`
- 修改了 `loadUserData()` 函数
- 优先显示 `user.id`，其次显示 `user.studentId`
```javascript
document.getElementById('idBadge').textContent = `ID: ${user.id || user.studentId || 'N/A'}`;
```

### 数据来源
- `user.id`: 后台自动生成的唯一 ID（S001, S002...）
- `user.studentId`: 学生的学号（如 2021211001）

## API 接口

### 更新个人资料
**端点**: `PUT /api/student/profile`

**请求体**:
```json
{
  "avatar": "data:image/png;base64,iVBORw0KGgoAAAANS...",
  "gpa": 3.75,
  "phone": "13800138000",
  "major": "Computer Science",
  "grade": "大三"
}
```

**响应**:
```json
{
  "code": 200,
  "message": "更新成功",
  "data": {
    "id": "S001",
    "name": "张三",
    "email": "zhangsan@bupt.edu.cn",
    "studentId": "2021211001",
    "avatar": "data:image/png;base64,iVBORw0KGgoAAAANS...",
    "gpa": 3.75,
    "major": "Computer Science",
    "grade": "大三"
  }
}
```

## 已知问题

### 1. 头像未保存到后端
**状态**: 待实现  
**描述**: 当前头像只保存在 localStorage 中，刷新后会保留，但未调用后端 API 保存到数据库。  
**解决方案**: 需要在 `initAvatarUpload()` 函数中添加 API 调用：
```javascript
await API.student.updateProfile({ avatar: base64Image });
```

### 2. GPA 未保存到后端
**状态**: 待实现  
**描述**: 当前 GPA 只保存在 localStorage 中。  
**解决方案**: 需要在 `initGPAModal()` 函数中添加 API 调用：
```javascript
await API.student.updateProfile({ gpa: gpa });
```

### 3. 头像文件大小
**状态**: 正常  
**描述**: Base64 编码会增加约 33% 的文件大小，2MB 的图片会变成约 2.7MB 的字符串。  
**建议**: 考虑使用图片压缩或上传到云存储服务。
