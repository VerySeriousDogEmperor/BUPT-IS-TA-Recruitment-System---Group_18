# 完整功能清单 - 第1部分：基础架构

## 1. 嵌入式服务器

### 功能作用
提供 HTTP 服务，处理前端请求，路由到相应的 Servlet，返回响应。替代传统的 Tomcat 部署方式，简化开发和部署流程。

### 涉及文件
- `src/EmbeddedServer.java` - 主服务器类
- `src/HttpServletRequestAdapter.java` - Request 适配器
- `src/HttpServletResponseAdapter.java` - Response 适配器
- `src/HttpSessionAdapter.java` - Session 适配器
- `QUICK-START.bat` - 启动脚本

### 数据链路
```
客户端请求
  ↓
JDK HttpServer (端口 9191)
  ↓
ServletAdapter (路由匹配)
  ↓
HttpServletRequestAdapter (请求适配)
  ↓
具体 Servlet (业务处理)
  ↓
HttpServletResponseAdapter (响应适配)
  ↓
客户端响应
```

### 设计思路
**适配器模式**：将 JDK 内置的 `HttpServer` 适配为标准的 Servlet 容器
- 使用 `HttpExchange` 封装请求/响应
- 实现 `HttpServletRequest/Response` 接口
- 支持 Session 管理
- 支持路径匹配和参数解析

### 实现逻辑
1. **服务器启动**
   - 创建 `HttpServer` 实例，绑定端口 9191
   - 注册所有 API 路由（`/api/auth`, `/api/jobs` 等）
   - 每个路由关联一个 `ServletAdapter` 和对应的 Servlet

2. **请求处理**
   - `HttpServer` 接收请求
   - 根据 URL 路径匹配到对应的 `ServletAdapter`
   - `ServletAdapter` 创建适配器对象
   - 调用 Servlet 的 `service()` 方法
   - 返回响应

3. **适配器实现**
   - `HttpServletRequestAdapter`：实现 `getPathInfo()`, `getParameter()`, `getSession()` 等方法
   - `HttpServletResponseAdapter`：实现 `getWriter()`, `setStatus()`, `setHeader()` 等方法
   - `HttpSessionAdapter`：实现 `getAttribute()`, `setAttribute()` 等方法

## 2. DDD 架构

### 功能作用
采用领域驱动设计（Domain-Driven Design）架构，将业务逻辑清晰分层，提高代码可维护性和可扩展性。

### 涉及文件
```
src/com/bupt/ta/
├── shared/              # 共享层
│   ├── domain/          # 领域实体
│   ├── infrastructure/  # 基础设施（数据访问）
│   ├── interfaces/      # 接口层（API）
│   └── util/            # 工具类
├── student/             # 学生端
│   ├── domain/
│   └── interfaces/
├── mo/                  # MO 端
│   └── interfaces/
└── admin/               # Admin 端
    └── interfaces/
```

### 数据链路
```
前端请求
  ↓
Interfaces 层 (Servlet)
  ↓
Application 层 (业务逻辑)
  ↓
Domain 层 (领域模型)
  ↓
Infrastructure 层 (Repository)
  ↓
DataStore (JSON 文件)
```

### 设计思路
**分层架构 + 领域驱动设计**
- **Interfaces 层**：处理 HTTP 请求，参数验证，调用业务逻辑
- **Application 层**：协调多个领域对象，实现业务用例
- **Domain 层**：核心业务逻辑和规则，领域实体
- **Infrastructure 层**：技术实现，数据持久化

### 实现逻辑
1. **领域实体**（Domain）
   - `User`, `Student`, `Job`, `Application`, `Timesheet` 等
   - 包含业务属性和行为
   - 纯 POJO，不依赖框架

2. **仓储模式**（Repository）
   - 封装数据访问逻辑
   - 提供 CRUD 操作
   - 使用 `DataStore` 统一管理数据

3. **接口层**（Servlet）
   - 继承 `BaseServlet`
   - 处理 HTTP 方法（GET, POST, PUT, DELETE）
   - 调用 Repository 进行数据操作
   - 使用 `ResponseUtil` 统一响应格式

## 3. JSON 文件存储

### 功能作用
使用 JSON 文件作为数据持久化方案，简化开发，无需配置数据库，数据可读性强，便于调试。

### 涉及文件
- `src/com/bupt/ta/shared/infrastructure/DataStore.java` - 数据存储管理器
- `src/com/bupt/ta/shared/util/JsonFileUtil.java` - JSON 文件工具
- `resources/data/*.json` - 数据文件
  - `students.json` - 学生数据
  - `users.json` - MO/Admin 数据
  - `jobs.json` - 职位数据
  - `applications.json` - 申请数据
  - `timesheets.json` - 工时数据
  - `modules.json` - 课程模块数据

### 数据链路
```
Repository
  ↓
DataStore.loadXxx()
  ↓
JsonFileUtil.readJson()
  ↓
Gson 反序列化
  ↓
List<Entity>
  ↓
业务处理
  ↓
DataStore.saveXxx()
  ↓
JsonFileUtil.writeJson()
  ↓
Gson 序列化
  ↓
写入 JSON 文件
```

### 设计思路
**仓储模式 + 读写锁**
- 使用 `ReadWriteLock` 保证并发安全
- 读操作：多线程可同时读取
- 写操作：独占锁，确保数据一致性
- 使用 Gson 进行 JSON 序列化/反序列化

### 实现逻辑
1. **数据加载**
   ```java
   public static List<Student> loadStudents() throws IOException {
       studentsLock.readLock().lock();
       try {
           Type type = new TypeToken<List<Student>>(){}.getType();
           List<Student> students = JsonFileUtil.readJson(STUDENTS_FILE, type);
           return students != null ? students : new ArrayList<>();
       } finally {
           studentsLock.readLock().unlock();
       }
   }
   ```

2. **数据保存**
   ```java
   public static void saveStudents(List<Student> students) throws IOException {
       studentsLock.writeLock().lock();
       try {
           JsonFileUtil.writeJson(STUDENTS_FILE, students);
       } finally {
           studentsLock.writeLock().unlock();
       }
   }
   ```

3. **Repository 操作**
   - `findAll()` - 加载所有数据
   - `findById()` - 根据 ID 查找
   - `save()` - 保存或更新（先加载，修改，再保存）
   - `delete()` - 删除（先加载，过滤，再保存）

## 4. 统一响应格式

### 功能作用
统一所有 API 的响应格式，便于前端处理，提供一致的错误处理机制。

### 涉及文件
- `src/com/bupt/ta/shared/util/ResponseUtil.java`

### 数据链路
```
Servlet 业务逻辑
  ↓
ResponseUtil.sendSuccess() / sendError()
  ↓
构建 JSON 响应
  ↓
设置 HTTP 状态码和 Content-Type
  ↓
写入响应流
  ↓
客户端接收
```

### 设计思路
**统一响应格式**
```json
{
  "code": 200,
  "message": "成功",
  "data": { ... }
}
```

### 实现逻辑
```java
public static void sendSuccess(HttpServletResponse response, String message, Object data) {
    Map<String, Object> result = new HashMap<>();
    result.put("code", 200);
    result.put("message", message);
    result.put("data", data);
    
    response.setContentType("application/json; charset=UTF-8");
    response.setStatus(200);
    
    Gson gson = new Gson();
    String json = gson.toJson(result);
    
    PrintWriter writer = response.getWriter();
    writer.write(json);
    writer.flush();
}
```

## 5. Session 管理

### 功能作用
管理用户登录状态，提供用户身份验证，支持多用户并发访问。

### 涉及文件
- `src/com/bupt/ta/shared/util/SessionUtil.java`
- `src/HttpSessionAdapter.java`

### 数据链路
```
用户登录
  ↓
AuthServlet 验证成功
  ↓
SessionUtil.setCurrentUser/Student()
  ↓
存储到 HttpSession
  ↓
后续请求携带 Session
  ↓
SessionUtil.getCurrentUser/Student()
  ↓
从 HttpSession 获取用户信息
```

### 设计思路
**Session 存储用户信息**
- 登录成功后，将用户对象存入 Session
- 后续请求从 Session 获取用户信息
- 支持学生和 MO/Admin 两种用户类型

### 实现逻辑
```java
// 设置当前学生
public static void setCurrentStudent(HttpServletRequest request, Student student) {
    HttpSession session = request.getSession(true);
    session.setAttribute("currentStudent", student);
    session.setAttribute("userType", "student");
}

// 获取当前学生
public static Student getCurrentStudent(HttpServletRequest request) {
    HttpSession session = request.getSession(false);
    if (session == null) return null;
    return (Student) session.getAttribute("currentStudent");
}
```
