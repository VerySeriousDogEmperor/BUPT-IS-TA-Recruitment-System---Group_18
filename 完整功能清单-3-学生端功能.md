# 完整功能清单 - 第3部分：学生端功能

## 10. 职位浏览

### 功能作用
学生可以浏览所有已发布的 TA 职位，支持搜索和筛选，查看职位详情。

### 涉及文件
**后端**
- `src/com/bupt/ta/shared/interfaces/JobServlet.java`
- `src/com/bupt/ta/shared/infrastructure/JobRepository.java`
- `src/com/bupt/ta/shared/domain/Job.java`

**前端**
- `web/index.html` - 主页职位列表
- `web/job-detail.html` - 职位详情页
- `web/static/js/utils/api.js`

### 数据链路
```
前端请求
  ↓
API.jobs.getList(params)
  ↓
GET /api/jobs?status=published&keyword=xxx
  ↓
JobServlet.doGet()
  ↓
JobRepository.findAll()
  ↓
DataStore.loadJobs()
  ↓
读取 jobs.json
  ↓
过滤：status = 'published'
  ↓
搜索：title/moduleCode 包含关键词
  ↓
返回职位列表
  ↓
前端渲染
```

### 设计思路
**公开接口 + 状态过滤**
- 不需要登录即可浏览
- 只显示已发布的职位（status = 'published'）
- 支持关键词搜索
- 支持按模块代码筛选

### 实现逻辑

#### 后端处理
```java
@Override
protected void doGet(HttpServletRequest request, HttpServletResponse response) {
    String pathInfo = request.getPathInfo();
    
    if (pathInfo == null || "/".equals(pathInfo)) {
        // 获取职位列表
        String keyword = request.getParameter("keyword");
        String moduleCode = request.getParameter("moduleCode");
        
        List<Job> jobs = jobRepo.findAll();
        
        // 只显示已发布的职位
        jobs = jobs.stream()
                .filter(job -> "published".equals(job.getStatus()))
                .collect(Collectors.toList());
        
        // 关键词搜索
        if (keyword != null && !keyword.isEmpty()) {
            jobs = jobs.stream()
                    .filter(job -> job.getTitle().contains(keyword) || 
                                   job.getModuleCode().contains(keyword))
                    .collect(Collectors.toList());
        }
        
        // 按模块筛选
        if (moduleCode != null && !moduleCode.isEmpty()) {
            jobs = jobs.stream()
                    .filter(job -> moduleCode.equals(job.getModuleCode()))
                    .collect(Collectors.toList());
        }
        
        ResponseUtil.sendSuccess(response, "获取成功", jobs);
    }
}
```

## 11. 职位申请

### 功能作用
学生可以申请感兴趣的 TA 职位，提交个人陈述，系统记录申请状态。

### 涉及文件
**后端**
- `src/com/bupt/ta/student/interfaces/StudentApplicationServlet.java`
- `src/com/bupt/ta/shared/infrastructure/ApplicationRepository.java`
- `src/com/bupt/ta/shared/domain/Application.java`

**前端**
- `web/apply.html` - 申请页面
- `web/student/dashboard.html` - 学生仪表板

### 数据链路
```
前端提交申请
  ↓
API.student.applyJob(jobId)
  ↓
POST /api/student/applications
  ↓
StudentApplicationServlet.doPost()
  ↓
验证：用户已登录
  ↓
验证：职位存在且已发布
  ↓
验证：未重复申请
  ↓
生成申请 ID
  ↓
创建 Application 对象
  - studentId: 当前学生 ID
  - jobId: 职位 ID
  - status: 'pending'
  - personalStatement: 个人陈述
  ↓
ApplicationRepository.save()
  ↓
写入 applications.json
  ↓
返回申请信息
```

### 设计思路
**一人一职位一申请**
- 验证学生未重复申请同一职位
- 初始状态为 pending（待审核）
- 记录申请时间和个人陈述
- 支持查看申请历史

### 实现逻辑

#### 后端处理
```java
@Override
protected void doPost(HttpServletRequest request, HttpServletResponse response) {
    // 1. 验证登录
    Student currentStudent = SessionUtil.getCurrentStudent(request);
    if (currentStudent == null) {
        ResponseUtil.sendError(response, 401, "未登录");
        return;
    }
    
    // 2. 读取申请数据
    Map<String, String> requestData = readRequestBody(request, Map.class);
    String jobId = requestData.get("jobId");
    String personalStatement = requestData.get("personalStatement");
    
    // 3. 验证职位存在
    Optional<Job> jobOpt = jobRepo.findById(jobId);
    if (!jobOpt.isPresent() || !"published".equals(jobOpt.get().getStatus())) {
        ResponseUtil.sendError(response, 404, "职位不存在或未发布");
        return;
    }
    
    // 4. 检查是否已申请
    List<Application> existing = applicationRepo.findByStudentAndJob(
        currentStudent.getId(), jobId);
    if (!existing.isEmpty()) {
        ResponseUtil.sendError(response, 400, "已申请过该职位");
        return;
    }
    
    // 5. 创建申请
    String applicationId = applicationRepo.generateId();
    Application application = new Application();
    application.setId(applicationId);
    application.setStudentId(currentStudent.getId());
    application.setJobId(jobId);
    application.setPersonalStatement(personalStatement);
    application.setStatus("pending");
    application.setAppliedAt(LocalDateTime.now());
    
    // 6. 保存
    applicationRepo.save(application);
    
    ResponseUtil.sendSuccess(response, "申请成功", application);
}
```

## 12. 查看申请状态

### 功能作用
学生可以查看自己的所有申请记录，包括申请状态、审核结果、审核意见。

### 涉及文件
**后端**
- `src/com/bupt/ta/student/interfaces/StudentApplicationServlet.java`
- `src/com/bupt/ta/shared/infrastructure/ApplicationRepository.java`

**前端**
- `web/student/dashboard.html` - 学生仪表板

### 数据链路
```
前端请求
  ↓
API.student.getApplications()
  ↓
GET /api/student/applications
  ↓
StudentApplicationServlet.doGet()
  ↓
验证：用户已登录
  ↓
ApplicationRepository.findByStudent(studentId)
  ↓
过滤：studentId = 当前学生 ID
  ↓
补充职位信息（JobRepository.findById）
  ↓
返回申请列表（含职位详情）
```

### 设计思路
**关联查询**
- 查询学生的所有申请
- 关联查询职位信息
- 显示申请状态和审核结果
- 按申请时间倒序排列

### 实现逻辑
```java
@Override
protected void doGet(HttpServletRequest request, HttpServletResponse response) {
    Student currentStudent = SessionUtil.getCurrentStudent(request);
    if (currentStudent == null) {
        ResponseUtil.sendError(response, 401, "未登录");
        return;
    }
    
    // 获取学生的所有申请
    List<Application> applications = applicationRepo.findAll().stream()
            .filter(app -> currentStudent.getId().equals(app.getStudentId()))
            .collect(Collectors.toList());
    
    // 补充职位信息
    List<Map<String, Object>> result = new ArrayList<>();
    for (Application app : applications) {
        Map<String, Object> item = new HashMap<>();
        item.put("application", app);
        
        // 添加职位信息
        Optional<Job> jobOpt = jobRepo.findById(app.getJobId());
        if (jobOpt.isPresent()) {
            item.put("job", jobOpt.get());
        }
        
        result.add(item);
    }
    
    ResponseUtil.sendSuccess(response, "获取成功", result);
}
```

## 13. 撤回申请

### 功能作用
学生可以撤回待审核的申请，已审核的申请不能撤回。

### 涉及文件
**后端**
- `src/com/bupt/ta/student/interfaces/StudentApplicationServlet.java`

**前端**
- `web/student/dashboard.html`

### 数据链路
```
前端点击撤回
  ↓
API.student.withdrawApplication(applicationId)
  ↓
DELETE /api/student/applications/{id}
  ↓
StudentApplicationServlet.doDelete()
  ↓
验证：用户已登录
  ↓
验证：申请存在
  ↓
验证：申请属于当前学生
  ↓
验证：申请状态为 pending
  ↓
ApplicationRepository.delete(applicationId)
  ↓
从 applications.json 删除
  ↓
返回成功
```

### 设计思路
**只能撤回待审核申请**
- 验证申请所有权
- 只允许撤回 pending 状态的申请
- 已审核（accepted/rejected）的不能撤回

### 实现逻辑
```java
@Override
protected void doDelete(HttpServletRequest request, HttpServletResponse response) {
    Student currentStudent = SessionUtil.getCurrentStudent(request);
    if (currentStudent == null) {
        ResponseUtil.sendError(response, 401, "未登录");
        return;
    }
    
    String pathInfo = request.getPathInfo();
    String applicationId = pathInfo.substring(1);
    
    // 验证申请存在
    Optional<Application> appOpt = applicationRepo.findById(applicationId);
    if (!appOpt.isPresent()) {
        ResponseUtil.sendError(response, 404, "申请不存在");
        return;
    }
    
    Application app = appOpt.get();
    
    // 验证所有权
    if (!currentStudent.getId().equals(app.getStudentId())) {
        ResponseUtil.sendError(response, 403, "无权操作");
        return;
    }
    
    // 只能撤回待审核的申请
    if (!"pending".equals(app.getStatus())) {
        ResponseUtil.sendError(response, 400, "只能撤回待审核的申请");
        return;
    }
    
    // 删除申请
    applicationRepo.delete(applicationId);
    
    ResponseUtil.sendSuccess(response, "撤回成功", null);
}
```

## 14. 工时表提交

### 功能作用
学生可以提交工作时间记录，包括工作日期、工时、工作描述，等待 MO 审核。

### 涉及文件
**后端**
- `src/com/bupt/ta/student/interfaces/StudentTimesheetServlet.java`
- `src/com/bupt/ta/shared/infrastructure/TimesheetRepository.java`
- `src/com/bupt/ta/shared/domain/Timesheet.java`

**前端**
- `web/student/dashboard.html` - 提交工时表

### 数据链路
```
前端提交工时表
  ↓
API.student.submitTimesheet(data)
  ↓
POST /api/student/timesheets
  ↓
StudentTimesheetServlet.doPost()
  ↓
验证：用户已登录
  ↓
验证：申请存在且已录用
  ↓
生成工时表 ID
  ↓
创建 Timesheet 对象
  - applicationId: 申请 ID
  - workDate: 工作日期
  - hoursWorked: 工作时长
  - description: 工作描述
  - status: 'pending'
  ↓
TimesheetRepository.save()
  ↓
写入 timesheets.json
  ↓
返回工时表信息
```

### 设计思路
**基于申请的工时记录**
- 必须有已录用的申请才能提交工时
- 记录工作日期、时长、描述
- 初始状态为 pending（待审核）
- 支持查看历史工时记录

### 实现逻辑
```java
@Override
protected void doPost(HttpServletRequest request, HttpServletResponse response) {
    Student currentStudent = SessionUtil.getCurrentStudent(request);
    if (currentStudent == null) {
        ResponseUtil.sendError(response, 401, "未登录");
        return;
    }
    
    // 读取工时数据
    Timesheet timesheet = readRequestBody(request, Timesheet.class);
    
    // 验证申请存在且已录用
    Optional<Application> appOpt = applicationRepo.findById(timesheet.getApplicationId());
    if (!appOpt.isPresent()) {
        ResponseUtil.sendError(response, 404, "申请不存在");
        return;
    }
    
    Application app = appOpt.get();
    if (!currentStudent.getId().equals(app.getStudentId())) {
        ResponseUtil.sendError(response, 403, "无权操作");
        return;
    }
    
    if (!"accepted".equals(app.getStatus())) {
        ResponseUtil.sendError(response, 400, "只有已录用的申请才能提交工时");
        return;
    }
    
    // 生成 ID 并保存
    String timesheetId = timesheetRepo.generateId();
    timesheet.setId(timesheetId);
    timesheet.setStatus("pending");
    timesheet.setSubmittedAt(LocalDateTime.now());
    
    timesheetRepo.save(timesheet);
    
    ResponseUtil.sendSuccess(response, "提交成功", timesheet);
}
```
