## 19. JSON 文件存储系统

### 功能作用
使用 JSON 文件作为数据持久化方案，提供统一的数据访问接口，支持并发读写，无需配置数据库。

### 涉及文件
**核心文件**
- `src/com/bupt/ta/shared/infrastructure/DataStore.java` - 数据存储管理器
- `src/com/bupt/ta/shared/util/JsonFileUtil.java` - JSON 文件读写工具

**数据文件**
- `resources/data/students.json` - 学生数据
- `resources/data/users.json` - MO/Admin 数据
- `resources/data/jobs.json` - 职位数据
- `resources/data/applications.json` - 申请数据
- `resources/data/timesheets.json` - 工时数据
- `resources/data/modules.json` - 课程模块数据

### 数据链路
```
Repository 层
  ↓
DataStore.loadXxx()
  ↓
JsonFileUtil.readJson()
  ↓
Gson.fromJson()
  ↓
List<Entity> (内存对象)
  ↓
业务处理（增删改查）
  ↓
DataStore.saveXxx()
  ↓
JsonFileUtil.writeJson()
  ↓
Gson.toJson()
  ↓
写入 JSON 文件
```

### 设计思路
**读写锁 + 统一管理**
- 使用 `ReadWriteLock` 保证线程安全
- 读操作：多线程可同时读取（共享锁）
- 写操作：独占锁，确保数据一致性
- 使用 Gson 进行 JSON 序列化/反序列化
- 统一的数据访问接口

### 实现逻辑

#### 读写锁机制
```java
// 每个数据文件一个读写锁
private static final ReadWriteLock studentsLock = new ReentrantReadWriteLock();
private static final ReadWriteLock usersLock = new ReentrantReadWriteLock();
private static final ReadWriteLock jobsLock = new ReentrantReadWriteLock();
// ...

// 读操作（共享锁）
public static List<Student> loadStudents() throws IOException {
    studentsLock.readLock().lock();
    try {
        Type type = new TypeToken<List<Student>>(){}.getType();
        List<Student> students = JsonFileUtil.readJson(STUDENTS_FILE, type);
        return students != null ? students : new ArrayList<>();
    } finally {
        studentsLock.readLock().unlock();
    }
}

// 写操作（独占锁）
public static void saveStudents(List<Student> students) throws IOException {
    studentsLock.writeLock().lock();
    try {
        JsonFileUtil.writeJson(STUDENTS_FILE, students);
    } finally {
        studentsLock.writeLock().unlock();
    }
}
```

#### JSON 序列化配置
```java
// JsonFileUtil.java
private static final Gson gson = new GsonBuilder()
        .setPrettyPrinting() // 格式化输出
        .registerTypeAdapter(LocalDateTime.class, new LocalDateTimeAdapter())
        .create();

// 读取 JSON
public static <T> T readJson(String filePath, Type type) throws IOException {
    File file = new File(filePath);
    if (!file.exists()) {
        return null;
    }
    
    try (Reader reader = new FileReader(file, StandardCharsets.UTF_8)) {
        return gson.fromJson(reader, type);
    }
}

// 写入 JSON
public static void writeJson(String filePath, Object data) throws IOException {
    File file = new File(filePath);
    file.getParentFile().mkdirs(); // 确保目录存在
    
    try (Writer writer = new FileWriter(file, StandardCharsets.UTF_8)) {
        gson.toJson(data, writer);
    }
}
```

## 20. Repository 模式

### 功能作用
封装数据访问逻辑，提供统一的 CRUD 操作接口，隔离业务逻辑和数据存储细节。

### 涉及文件
- `src/com/bupt/ta/shared/infrastructure/StudentRepository.java`
- `src/com/bupt/ta/shared/infrastructure/UserRepository.java`
- `src/com/bupt/ta/shared/infrastructure/JobRepository.java`
- `src/com/bupt/ta/shared/infrastructure/ApplicationRepository.java`
- `src/com/bupt/ta/shared/infrastructure/TimesheetRepository.java`
- `src/com/bupt/ta/shared/infrastructure/ModuleRepository.java`

### 数据链路
```
Servlet 调用 Repository
  ↓
Repository.findById(id)
  ↓
DataStore.loadXxx()
  ↓
遍历查找匹配的对象
  ↓
返回 Optional<Entity>
  ↓
Servlet 处理业务逻辑
  ↓
Repository.save(entity)
  ↓
DataStore.loadXxx() (加载所有数据)
  ↓
查找并更新或添加
  ↓
DataStore.saveXxx() (保存所有数据)
```

### 设计思路
**仓储模式 + Optional 返回值**
- 提供统一的 CRUD 接口
- 使用 `Optional` 处理查询结果
- 自动生成递增 ID
- 支持条件查询（findByEmail, findByStudentId 等）

### 实现逻辑

#### 基本 CRUD 操作
```java
public class StudentRepository {
    // 查询所有
    public List<Student> findAll() throws IOException {
        return DataStore.loadStudents();
    }
    
    // 根据 ID 查询
    public Optional<Student> findById(String id) throws IOException {
        List<Student> students = DataStore.loadStudents();
        return students.stream()
                .filter(s -> id.equals(s.getId()))
                .findFirst();
    }
    
    // 保存或更新
    public void save(Student student) throws IOException {
        List<Student> students = DataStore.loadStudents();
        
        // 查找是否已存在
        boolean found = false;
        for (int i = 0; i < students.size(); i++) {
            if (students.get(i).getId().equals(student.getId())) {
                students.set(i, student); // 更新
                found = true;
                break;
            }
        }
        
        if (!found) {
            students.add(student); // 新增
        }
        
        DataStore.saveStudents(students);
    }
    
    // 删除
    public void delete(String id) throws IOException {
        List<Student> students = DataStore.loadStudents();
        students.removeIf(s -> id.equals(s.getId()));
        DataStore.saveStudents(students);
    }
}
```

#### 自动生成 ID
```java
public String generateId() throws IOException {
    List<Student> students = DataStore.loadStudents();
    int maxId = 0;
    
    // 找到最大的 ID 数字
    for (Student s : students) {
        if (s.getId().startsWith("S")) {
            try {
                int id = Integer.parseInt(s.getId().substring(1));
                if (id > maxId) maxId = id;
            } catch (NumberFormatException e) {
                // 忽略非数字 ID
            }
        }
    }
    
    // 返回下一个 ID
    return String.format("S%03d", maxId + 1); // S001, S002, S003...
}
```

#### 条件查询
```java
// 根据邮箱查询
public Optional<Student> findByEmail(String email) throws IOException {
    List<Student> students = DataStore.loadStudents();
    return students.stream()
            .filter(s -> email.equals(s.getEmail()))
            .findFirst();
}

// 根据学号查询
public Optional<Student> findByStudentId(String studentId) throws IOException {
    List<Student> students = DataStore.loadStudents();
    return students.stream()
            .filter(s -> studentId.equals(s.getStudentId()))
            .findFirst();
}

// 复杂查询：根据学生和职位查询申请
public List<Application> findByStudentAndJob(String studentId, String jobId) throws IOException {
    List<Application> applications = DataStore.loadApplications();
    return applications.stream()
            .filter(app -> studentId.equals(app.getStudentId()) && 
                          jobId.equals(app.getJobId()))
            .collect(Collectors.toList());
}
```

## 数据管理总结

### 核心特点
1. **无数据库依赖**：使用 JSON 文件存储，简化部署
2. **线程安全**：读写锁机制保证并发安全
3. **统一管理**：DataStore 提供统一的数据访问接口
4. **仓储模式**：Repository 封装数据访问逻辑
5. **自动 ID 生成**：支持递增 ID 自动生成
6. **完整测试数据**：DataInitializer 提供完整的测试数据集

### 数据流转
```
应用启动
  ↓
DataInitializer 初始化测试数据（可选）
  ↓
Servlet 接收请求
  ↓
Repository 查询数据
  ↓
DataStore 加载 JSON 文件
  ↓
业务处理
  ↓
Repository 保存数据
  ↓
DataStore 写入 JSON 文件
```

### 优缺点

#### 优点
- 无需配置数据库，部署简单
- 数据可读性强，便于调试
- 适合小型项目和原型开发
- 支持版本控制（Git）

#### 缺点
- 不适合大数据量
- 不支持复杂查询（需要加载全部数据）
- 并发性能有限
- 无事务支持

### 生产环境建议
- 迁移到关系型数据库（MySQL, PostgreSQL）
- 使用 ORM 框架（Hibernate, MyBatis）
- 添加数据库连接池
- 实现事务管理
- 添加数据备份机制
