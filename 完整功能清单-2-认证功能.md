# 完整功能清单 - 第2部分：认证功能

## 6. 用户登录

### 功能作用
验证用户身份，建立登录会话，支持学生、MO、Admin 三种角色登录。

### 涉及文件
**后端**
- `src/com/bupt/ta/shared/interfaces/AuthServlet.java` - 认证接口
- `src/com/bupt/ta/shared/infrastructure/StudentRepository.java` - 学生数据访问
- `src/com/bupt/ta/shared/infrastructure/UserRepository.java` - MO/Admin 数据访问
- `src/com/bupt/ta/shared/util/SessionUtil.java` - Session 管理

**前端**
- `web/login.html` - 登录页面
- `web/static/js/pages/login.js` - 登录逻辑
- `web/static/js/utils/api.js` - API 调用
- `web/static/js/utils/auth.js` - 认证工具

### 数据链路
```
前端表单提交
  ↓
API.auth.login(email, password, role)
  ↓
POST /api/auth/login
  ↓
AuthServlet.handleLogin()
  ↓
根据 role 查询数据库
  - student → StudentRepository.findByEmail()
  - mo/admin → UserRepository.findByEmail()
  ↓
验证密码
  ↓
SessionUtil.setCurrentUser/Student()
  ↓
更新最后登录时间
  ↓
返回用户信息（不含密码）
  ↓
前端存储到 localStorage
  ↓
跳转到对应页面
```

### 设计思路
**基于 Session 的认证**
- 使用 HTTP Session 存储登录状态
- 密码明文比对（生产环境需加密）
- 支持多角色登录（student, mo, admin）
- 前端使用 localStorage 缓存用户信息

### 实现逻辑

#### 后端处理
```java
private void handleLogin(HttpServletRequest request, HttpServletResponse response) {
    // 1. 读取请求参数
    LoginRequest loginReq = readRequestBody(request, LoginRequest.class);
    
    // 2. 验证参数
    if (loginReq.email == null || loginReq.password == null) {
        ResponseUtil.sendError(response, 400, "邮箱和密码不能为空");
        return;
    }
    
    // 3. 根据角色查找用户
    if ("student".equals(loginReq.role)) {
        Optional<Student> studentOpt = studentRepo.findByEmail(loginReq.email);
        if (!studentOpt.isPresent() || !student.getPassword().equals(loginReq.password)) {
            ResponseUtil.sendError(response, 400, "邮箱或密码错误");
            return;
        }
        
        // 4. 设置 Session
        SessionUtil.setCurrentStudent(request, student);
        
        // 5. 更新最后登录时间
        student.setLastLoginAt(LocalDateTime.now());
        studentRepo.save(student);
        
        // 6. 返回用户信息
        ResponseUtil.sendSuccess(response, "登录成功", userData);
    }
}
```

#### 前端处理
```javascript
async function handleSubmit(e) {
    e.preventDefault();
    
    const email = formData.get('email');
    const password = formData.get('password');
    
    // 调用登录 API
    const userData = await API.auth.login(email, password, 'student');
    
    // 存储登录信息
    localStorage.setItem('userRole', 'student');
    localStorage.setItem('isAuthenticated', 'true');
    localStorage.setItem('user', JSON.stringify(userData));
    
    // 跳转到主页
    window.location.href = '/index.html';
}
```

## 7. 用户注册

### 功能作用
允许学生注册新账号，验证邮箱和学号唯一性，自动生成用户 ID。

### 涉及文件
**后端**
- `src/com/bupt/ta/shared/interfaces/AuthServlet.java`
- `src/com/bupt/ta/shared/infrastructure/StudentRepository.java`
- `src/com/bupt/ta/student/domain/Student.java`

**前端**
- `web/login.html` - 注册表单
- `web/static/js/pages/login.js` - 注册逻辑

### 数据链路
```
前端注册表单
  ↓
API.auth.register(userData)
  ↓
POST /api/auth/register
  ↓
AuthServlet.handleRegister()
  ↓
验证邮箱唯一性 (StudentRepository.findByEmail)
  ↓
验证学号唯一性 (StudentRepository.findByStudentId)
  ↓
生成新 ID (StudentRepository.generateId)
  ↓
创建 Student 对象
  ↓
保存到数据库 (StudentRepository.save)
  ↓
写入 students.json
  ↓
返回用户信息
  ↓
前端自动登录
```

### 设计思路
**注册即登录**
- 注册成功后自动设置 Session
- 验证邮箱和学号唯一性
- 自动生成递增 ID（S001, S002...）
- 设置默认状态为 active

### 实现逻辑

#### 后端处理
```java
private void handleRegister(HttpServletRequest request, HttpServletResponse response) {
    // 1. 读取注册信息
    RegisterRequest regReq = readRequestBody(request, RegisterRequest.class);
    
    // 2. 验证邮箱唯一性
    if (studentRepo.findByEmail(regReq.email).isPresent()) {
        ResponseUtil.sendError(response, 400, "邮箱已被注册");
        return;
    }
    
    // 3. 验证学号唯一性
    if (studentRepo.findByStudentId(regReq.studentId).isPresent()) {
        ResponseUtil.sendError(response, 400, "学号已存在");
        return;
    }
    
    // 4. 生成新 ID
    String newId = studentRepo.generateId(); // S001, S002...
    
    // 5. 创建学生对象
    Student student = new Student();
    student.setId(newId);
    student.setEmail(regReq.email);
    student.setPassword(regReq.password);
    student.setName(regReq.name);
    student.setStudentId(regReq.studentId);
    student.setRole("student");
    student.setStatus("active");
    student.setCreatedAt(LocalDateTime.now());
    
    // 6. 保存
    studentRepo.save(student);
    
    // 7. 自动登录
    SessionUtil.setCurrentStudent(request, student);
    
    // 8. 返回用户信息
    ResponseUtil.sendSuccess(response, "注册成功", userData);
}
```

#### ID 生成逻辑
```java
public String generateId() throws IOException {
    List<Student> students = DataStore.loadStudents();
    int maxId = 0;
    
    // 找到最大的 ID 数字
    for (Student s : students) {
        if (s.getId().startsWith("S")) {
            int id = Integer.parseInt(s.getId().substring(1));
            if (id > maxId) maxId = id;
        }
    }
    
    // 返回下一个 ID
    return String.format("S%03d", maxId + 1); // S001, S002...
}
```

## 8. 用户登出

### 功能作用
清除用户登录状态，销毁 Session，清除前端缓存。

### 涉及文件
**后端**
- `src/com/bupt/ta/shared/interfaces/AuthServlet.java`

**前端**
- `web/index.html` - 主页登出按钮
- `web/static/js/utils/auth.js` - 认证工具

### 数据链路
```
前端点击 Logout
  ↓
API.auth.logout()
  ↓
POST /api/auth/logout
  ↓
AuthServlet.handleLogout()
  ↓
request.getSession().invalidate()
  ↓
清除 Session
  ↓
返回成功
  ↓
前端清除 localStorage
  ↓
刷新页面
```

### 设计思路
**双重清除**
- 服务器端：销毁 Session
- 客户端：清除 localStorage
- 确保登出后无法访问受保护资源

### 实现逻辑

#### 后端处理
```java
private void handleLogout(HttpServletRequest request, HttpServletResponse response) {
    HttpSession session = request.getSession(false);
    if (session != null) {
        session.invalidate(); // 销毁 Session
    }
    ResponseUtil.sendSuccess(response, "登出成功", null);
}
```

#### 前端处理
```javascript
async function handleLogout(event) {
    event.preventDefault();
    
    try {
        // 调用登出 API
        await API.auth.logout();
    } catch (error) {
        console.error('Logout error:', error);
    }
    
    // 清除本地存储
    localStorage.removeItem('isAuthenticated');
    localStorage.removeItem('userRole');
    localStorage.removeItem('user');
    
    // 刷新页面
    window.location.reload();
}
```

## 9. 获取当前用户信息

### 功能作用
获取当前登录用户的详细信息，用于前端显示用户资料。

### 涉及文件
**后端**
- `src/com/bupt/ta/shared/interfaces/AuthServlet.java`

**前端**
- `web/static/js/utils/api.js`

### 数据链路
```
前端请求
  ↓
API.auth.me()
  ↓
GET /api/auth/me
  ↓
AuthServlet.handleGetCurrentUser()
  ↓
SessionUtil.getCurrentStudent/User()
  ↓
从 Session 获取用户信息
  ↓
返回用户数据（不含密码）
```

### 设计思路
**基于 Session 的用户识别**
- 不需要传递用户 ID
- 从 Session 中获取当前用户
- 返回完整的用户信息（排除敏感字段）

### 实现逻辑
```java
private void handleGetCurrentUser(HttpServletRequest request, HttpServletResponse response) {
    // 尝试获取学生
    Student student = SessionUtil.getCurrentStudent(request);
    if (student != null) {
        Map<String, Object> userData = new HashMap<>();
        userData.put("id", student.getId());
        userData.put("name", student.getName());
        userData.put("email", student.getEmail());
        userData.put("role", student.getRole());
        // ... 其他字段
        ResponseUtil.sendSuccess(response, "获取成功", userData);
        return;
    }
    
    // 尝试获取 MO/Admin
    User user = SessionUtil.getCurrentUser(request);
    if (user != null) {
        // 返回用户信息
        ResponseUtil.sendSuccess(response, "获取成功", userData);
        return;
    }
    
    // 未登录
    ResponseUtil.sendError(response, 401, "未登录");
}
```
